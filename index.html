Sensor Node

A waterproof ultrasonic sensor (JSN-SR04T) is mounted on a pole above the water surface, pointing downward. It fires a 40kHz pulse, measures the echo return time, and calculates water depth. The ESP32 microcontroller wakes every few minutes, takes a reading, and transmits over LoRa radio — then goes back to sleep to preserve battery. No mains power. No cellular SIM.

ESP32 microcontroller JSN-SR04T ultrasonic Battery powered Solar option available ~$40–75 CAD/node
LoRaWAN Gateway

A single gateway receives transmissions from sensor nodes up to 2–10km away over LoRa radio — no cellular network needed. One gateway can serve dozens of sensor nodes across a catchment area. It forwards data over standard internet (ethernet or WiFi) to the cloud network server. If LoRa coverage is unavailable at a site, NB-IoT or standard LTE are fallback options using the same sensor hardware.

2–10 km radio range One gateway, many sensors No cellular required NB-IoT fallback available
Network Server

The Things Network (TTN) is a free, open LoRaWAN network server that receives, decodes, and routes sensor data. Alert logic runs here — configurable thresholds (absolute water level) and rate-of-rise detection (e.g. water rising faster than 5cm/min). Over time, readings can be correlated with Environment Canada rainfall data and local catchment characteristics to build predictive models that alert before water even begins to rise.

The Things Network (TTN) Threshold + rate-of-rise logic Weather API integration Cloud-hosted, free tier
Alert

When a threshold is breached, a webhook fires within seconds — delivering an alert to email, SMS, or an operations dashboard. The 30–60 minute early warning window (before overtopping) gives field crews time to inspect, close roads proactively, or deploy temporary measures. All data is logged for post-event analysis and model refinement.

What Can Be Detected

More than just
rising water.

A single upstream sensor gives you two distinct alert types — flood early warning and blockage detection — from the same hardware. The difference lies in how fast the water rises, and whether rainfall explains it.

Flood early warning
Water level rising with rainfall

During intense rainfall, upstream water backs up against the culvert as flow exceeds capacity. The sensor detects the rising surface and fires an alert 30–60 minutes before overtopping — giving operations teams time to respond before road damage occurs.

Threshold alert Rate-of-rise alert Weather correlated
Blockage detection
Water rising faster than rain explains

When debris blocks the culvert inlet, water backs up faster than rainfall alone would predict. If the sensor is mounted directly above the inlet, floating debris also reflects the ultrasonic pulse back sooner — giving a falsely high reading that triggers an alert. A blockage in dry weather is easy to clear. A blockage during a storm becomes a washout.

Anomalous rate-of-rise Debris surface reflection Dry-weather trigger
Phase 2: upstream + downstream sensors

Adding a second sensor downstream of the culvert allows direct flow differential measurement — if upstream is rising but downstream isn't, the culvert is blocked. This removes ambiguity and gives maintenance teams a precise, actionable signal even without rainfall context.

Benefits
  • Unambiguous blockage confirmation
  • Flow rate estimation across culvert
  • Richer dataset for ML model training
  • Useful even in dry conditions
Tradeoffs
  • ~2× hardware cost per culvert (~$150 CAD)
  • Two installation points per site
  • Two potential failure points to maintain
  • Downstream access can be difficult
Pilot Proposal

One site. Three months. No cost to Halifax Water.

The proposal is deliberately small — a single proof-of-concept deployment at a known problem culvert. Data is advisory only. No operational dependency. If it works, phase 2 follows naturally.

1 Months 1–3
Proof of concept

One sensor node at one candidate culvert identified by Halifax Water. Data streams to a shared dashboard. Simple threshold alerts. Hardware supplied at no cost. Focus: does the sensor work reliably in NS conditions?

Hardware: ~$150 CAD · No ongoing cost to HW
2 Months 4–9
Multi-site expansion

If phase 1 is successful, expand to 3–5 known problem sites. Introduce weather API correlation. Begin building per-culvert response curves. Explore joint grant application (Sustainable Communities Challenge Fund).

Est. cost: $500–800 CAD hardware · grant-fundable
3 Year 2+
Predictive intelligence

ML model trained on sensor + weather + historical flood data. Predictive alerts fire before water rises — based on rainfall forecast and catchment characteristics. System becomes a climate adaptation dataset for NS infrastructure.

Potential grant funding: $75k–200k CAD
